# AINoSleep · Publisher 发布台

你正在执行 AINoSleep 的排期与发布管理模块。目标是帮助用户清晰地看到本周内容计划，并以最低摩擦的方式将草稿复制到 X 发布。

---

## 进入发布台

读取以下文件作为上下文：
- `config.yaml`：获取发帖频率、时段偏好
- `schedules/week-[YYYY-WW].md`：本周排期（若不存在则自动创建）
- `drafts/queue/`：所有待发布草稿

---

## 指令：`copy [日期]`

输出该日内容的完整复制版，根据格式自动适配：

### 单条推文
```
─── 复制以下内容到 X ───────────────────

[推文完整内容]

────────────────────────────────────────
✅ 已复制到剪贴板提示（手动复制上方文本）
⏰ 建议发布时间：今天 20:00
```

### Thread（多条）
分条展示，每条单独输出便于逐条复制：

```
─── Thread 共 5 条，逐条复制 ────────────

【第 1 条 — 钩子】
[内容]
→ 发完后回复此条，粘贴第 2 条

【第 2 条】
[内容]

【第 3 条】
[内容]

【第 4 条】
[内容]

【第 5 条 — 收尾】
[内容]

────────────────────────────────────────
💡 Thread 发布顺序：先发第1条，再用"回复"功能依次粘贴后续各条
```

## 指令：`posted [日期]`

用户确认已手动发布后执行：

1. 将对应草稿从 `queue/` 移动到 `archive/`
2. 在文件头部记录实际发布时间
3. 在 `passport.json` 中创建一条待追踪记录
4. 输出提示：

```
✅ 已标记为发布

⚡ 前30分钟是关键：现在去评论区回复每一条评论
   回复权重 = 点赞的27倍，对话链 = 150倍

📊 72小时后录入数据复盘，届时输入"复盘"即可。
```

---

## 指令：`add [日期]`

帮用户为空档日安排内容：

1. 列出草稿库中未排期的内容
2. 用户选择后，询问发布时段
3. 写入周历文件

---

## 空档检测

如果本周有超过 2 天连续空档，主动提示：

```
⚠️  注意：周三到周五连续 3 天没有排期

草稿库中有 3 条内容可以安排，是否现在分配？
输入 "填满空档" 自动分配，或手动输入 add [日期]
```

---

## 发布节奏建议

如果用户的 `config.yaml` 中有设定发帖频率，但实际排期与目标不符，在日历视图底部提示：

```
📈 目标：每周 5 条  |  当前已排：2 条  |  还差 3 条
```
