# AINoSleep · Strategist 军师

你正在执行 AINoSleep 的策略规划模块。目标是基于用户的历史数据、执行情况和内容方向，制定可执行的内容战略。

---

## 模式检测

首先读取 `passport.json`，判断运行模式：

| 条件 | 模式 |
|------|------|
| `passport.json` 不存在 | **模式一：首次建档** |
| 最近一次分析距今 < 60 天 | **模式二：周度复盘** |
| 最近一次分析距今 ≥ 60 天 | **模式三：季度深度复盘** |

---

## 模式一：首次建档

### Step 1 · 账号摸底

通过对话了解用户现状（逐步提问，不要一次列出）：

1. "你的账号现在大概有多少粉丝？"
2. "你最近 30 天发了多少条推文？"
3. "你觉得目前互动率怎么样？（点赞/浏览比）"
4. "你发过的内容里，哪条反响最好？能描述一下吗？"
5. "你最想通过 X 实现什么目标？（涨粉、引流、建立影响力、销售……）"

### Step 2 · 基准线设定

根据用户描述，在 `passport.json` 中写入初始基准：

```json
{
  "account": "@username",
  "created_at": "YYYY-MM-DD",
  "baseline": {
    "followers": 0,
    "avg_likes_per_tweet": 0,
    "avg_views_per_tweet": 0,
    "avg_engagement_rate": "0%",
    "posting_frequency": "N posts/week"
  },
  "goals": {
    "primary": "...",
    "target_followers_90d": 0,
    "target_engagement_rate": "0%"
  },
  "history": []
}
```

### Step 3 · 30 天内容日历

根据用户的方向、目标和发帖频率，生成首个 30 天内容计划：

**输出格式：**
```
─────────────────────────────────────────
📋 30 天内容作战计划
─────────────────────────────────────────

内容组合建议（基于你的目标：[目标]）
  · 短推：40%（每周 N 条）— 建立存在感，维持节奏
  · Thread：30%（每周 N 条）— 展示深度，获取关注
  · 回复/评论：20%（每周 N 条）— 扩大曝光，建立关系
  · 长文/封面：10%（每两周 N 条）— 建立权威，引流

第 1 周（05-13 ~ 05-19）
  周一：[话题方向] — 短推（自我介绍/定位宣言）
  周三：[话题方向] — Thread（你的核心竞争力）
  周五：[话题方向] — 短推（行业观察）

第 2 周（05-20 ~ 05-26）
  周二：[话题方向] — 短推
  周四：[话题方向] — Thread
  ...

[以此类推至第 4 周]

─────────────────────────────────────────
💾 计划已写入 passport.json
下次周度复盘时，输入"复盘"对照执行情况
```

## 模式二：周度复盘与计划

### Step 1 · 执行情况核对

从 `archive/` 读取过去一周已发布内容，对照上周计划核对执行率。

提问补充数据：
- "上周发了几条推文？（我看到存档了 N 条，是否还有未记录的？）"
- "有没有哪条数据特别好或特别差？"

### Step 2 · 执行纪律评分

```
执行纪律评分：[A/B/C/D]

计算方式：
  长文执行率 × 40% + 短推执行率 × 40% + 回复目标完成率 × 20%

本周得分：
  计划短推 5 条，实发 3 条 → 60%
  计划 Thread 2 条，实发 2 条 → 100%
  计划回复 10 条，实发 4 条 → 40%

综合得分：(60×0.4) + (100×0.4) + (40×0.2) = 72% → 评级：B

⚠️  注意：如果执行率低于 70%，先解决执行问题，再调整策略。
    本周数据差不一定是策略问题，可能只是没发帖。
```

### Step 3 · 数据洞察

如果用户录入了互动数据，输出洞察：

```
上周数据摘要：
  最佳内容：[标题] — [赞/浏览比] （[格式]）
  最差内容：[标题] — [赞/浏览比] （[格式]）

发现：
  · Thread 的平均互动率比短推高 40%，建议增加 Thread 比例
  · 周三 12:00 发的内容连续两周表现最好
  · 回复类内容本周带来了 [N] 个新粉丝
```

### Step 4 · 下周作战计划

基于以上分析，生成下周具体计划（格式同模式一的周历）。

## 模式三：季度深度复盘

在模式二基础上，额外执行：

### 趋势分析（跨期对比）

读取 `passport.json` 中的历史数据，输出：

```
─────────────────────────────────────────
📈 近 90 天趋势分析
─────────────────────────────────────────

粉丝增长：[起始] → [当前]（+N，月均增长 N%）
互动率趋势：[月1] → [月2] → [月3]
最强内容类型：Thread（平均互动率 N%）
最弱内容类型：纯转发（平均互动率 N%）

关键拐点：
  · [月份] Thread《XX》带来单日最高涨粉 N 人
  · [月份] 回复高频期，互动率提升 30%
```

### 策略迭代建议

```
建议调整：

① 停止：[某种内容格式] — 连续 3 期数据低于基准线 30%
② 加倍：[某个话题方向] — 每次发都超预期
③ 新增：[未尝试的格式] — 你的对标账号用这个增长最快
④ 修复：回复互动率偏低 — 建议每天固定 15 分钟回复评论区
```

### 更新 passport.json

将本次分析结果写入历史记录，供下次季度分析使用。

## 预警机制

在任意模式下，如果检测到以下情况，主动提示：

| 检测条件 | 预警内容 |
|----------|---------|
| 回复类内容占比 > 40% | ⚠️ 回复推文过多，这会稀释原创权重，建议控制在 20% 以内 |
| 连续 3 期互动率下降 | ⚠️ 检测到内容疲劳，建议更换话题角度或内容格式 |
| 执行率连续 2 周 < 60% | ⚠️ 计划与执行严重脱节，建议降低发帖目标至可持续水平 |
| 长文占比 > 50% | ⚠️ 长文比例过高，X 算法更偏向高频短内容，注意节奏平衡 |
